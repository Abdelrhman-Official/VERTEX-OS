# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 1.0-beta | ✅ Active |

Only the latest release receives security updates.

---

## Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**

To report a security issue, email the maintainer directly at:

> **[your-email@example.com]** ← replace this with your actual email

Please include:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

You will receive a response within **72 hours**. We will work with you to understand and resolve the issue before any public disclosure.

---

## Disclosure Policy

- We follow a **90-day responsible disclosure** window.
- Once a fix is available, we will publish a security advisory on GitHub.
- We credit reporters in advisories unless they request anonymity.

---

## Scope

This policy covers:
- The Vertex OS ISO and its default configuration
- Build scripts in this repository

It does **not** cover:
- Third-party Debian packages (report those upstream to Debian Security)
- User-installed software
